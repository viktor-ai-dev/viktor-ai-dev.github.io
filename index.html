📝 AI Customer Support System – Architecture & Core Components

Overview

This system is an AI-powered customer support engine for e-commerce, combining RAG, hybrid search, AI routing, memory, and query rewriting to deliver accurate and context-aware responses.

1. Data Ingestion

Purpose: Enable understanding of business data like product catalogs, policies, and FAQs.

Implementation: Users upload .txt files.

content = await file.read()
text = content.decode("utf-8")
chunks = splitter.split_text(text)

2. Chunking Strategy

Purpose: Break large docs into smaller pieces for efficient retrieval and context understanding.

splitter = RecursiveCharacterTextSplitter(
    chunk_size=800,
    chunk_overlap=100
)

Improves retrieval accuracy and prevents fragmented context.

3. Hybrid Search (Vector + Keyword)

Combines semantic embeddings search with keyword matching for precise retrieval.

vector_docs = retriever.invoke(rewritten_question)
keyword_docs = [
    doc for doc in vector_docs
    if any(word in doc.page_content.lower() for word in rewritten_question.lower().split())
]
unique_docs = []
seen = set()
for doc in vector_docs + keyword_docs:
    if doc.page_content not in seen:
        unique_docs.append(doc)
        seen.add(doc.page_content)
top_docs = sorted(unique_docs, key=lambda d: score(d), reverse=True)[:5]

4. LLM Orchestration

Components:

  • Prompt Engineering: Guides LLM to answer using context only.
  • AI Routing: Classifies user query (products, policy, faq).
  • Query Rewriting: Clarifies vague questions before retrieval.
  • Conversation Memory: Maintains context across messages.

5. Full System Flow

User Question → Query Rewriting → AI Routing → Hybrid Search → Context Building → LLM Response → Save to Memory

6. Key Takeaways

  • RAG is about both data structure and prompt design
  • Hybrid search improves retrieval accuracy
  • Query rewriting helps understand vague input
  • Memory enables natural conversation flow
  • AI routing replaces rigid rule-based systems
